# REQ-36: Configurator UX Redesign — единый стиль и явный выбор режима

**Статус:** Approved  
**Приоритет:** P1  
**Компонент:** configurator | ui | i18n  

## Описание

Текущий конфигуратор использует несколько несовместимых стилей карточек выбора:
- `StepWaterSource`: 2-кол/3-кол grid, иконка + название, нет описания
- `StepTestingStatus`: 2-кол grid, иконка + название + описание
- `StepProblems` (симптомы): 2-кол/4-кол grid, только иконка + название
- `StepUse`: 2-кол/4-кол grid, иконка + название + цвет, нет описания
- `StepScope`: 2-кол/3-кол grid, иконка + название, нет описания

Кроме этого, кнопка Advanced Mode — крошечный `text-xs`-элемент в углу шага 3, незаметный и никак не связанный с выбором "Laboratory analysis" на предыдущем шаге.

### Что нужно сделать

#### 1. Единый компонент выбора — RadioOptionList

Заменить все grid-карточки на единый паттерн: **горизонтальная строка** с иконкой слева, текстом (заголовок + подзаголовок) в центре и чекмарком справа. Применить ко всем шагам одиночного выбора:

```
┌──────────────────────────────────────────────────┐
│  [Icon]  Title                              [✓]  │
│          Subtitle description                    │
└──────────────────────────────────────────────────┘
```

- **StepWaterSource** — 6 источников, radio-строки, добавить краткие описания в i18n
- **StepTestingStatus** — 4 варианта, radio-строки (описания уже есть)
- **StepUse** — 7 вариантов, radio-строки, добавить краткие описания в i18n
- **StepScope** — 5 вариантов, radio-строки, добавить краткие описания в i18n

Для мультивыбора (симптомы в `StepProblems`) — те же строки, но с checkbox-индикатором вместо radio.

Поведение при клике: **auto-advance** остаётся (клик по строке → выбрать + через 180ms перейти дальше) — как сейчас, только визуально унифицировано.

#### 2. Явный выбор режима на шаге 3

Перед показом симптомов или списка контаминантов — вставить экран выбора пути. Два варианта radio-строками:

```
┌──────────────────────────────────────────────────┐
│  🔬  I have a lab report                         │
│      Select contaminants from your analysis      │
├──────────────────────────────────────────────────┤
│  💬  Describe by symptoms                        │
│      Tell us what you notice, smell, or taste    │
└──────────────────────────────────────────────────┘
```

- Клик "I have a lab report" → переключить wizardMode в 'advanced', перейти к `StepAdvancedContaminants`
- Клик "Describe by symptoms" → переключить wizardMode в 'simple', перейти к симптомам
- Убрать tiny toggle-кнопку из угла шага 3 полностью

#### 3. Удалить маленькую кнопку переключения режима

Текущий код в `Wizard.tsx`:
```tsx
<button onClick={() => switchMode('advanced')} className="... text-xs text-slate-500 ...">
  <Microscope size={12} />
  {t('configurator.advancedMode')}
</button>
```
— убрать. Выбор режима теперь происходит на явном экране выбора пути.

### Архитектура

- Создать переиспользуемый компонент `RadioOptionList` (или `OptionRow`) в `src/components/configurator/`
- Типизация: `{ id: string; icon: LucideIcon; color?: string; label: string; description?: string }`
- Анимация hover: `scale: 1.01` или убрать (при hover на строку, не карточку)
- Стиль выбранной строки: `bg-sky-500/10 border-l-2 border-sky-500 text-white`
