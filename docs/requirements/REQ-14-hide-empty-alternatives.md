# REQ-14: Скрывать пустые тиры в «Other Options»

**Статус:** Approved  
**Приоритет:** P1  
**Компонент:** configurator  

## Описание

Если движок не смог подобрать ни одного модуля для тира (например, бюджет $50 слишком мал для любого из подходящих фильтров), такой тир сейчас всё равно попадает в блок «Other Options» и показывает ошибку «No modules fit within this budget». Это вводит пользователя в заблуждение — незачем предлагать «систему» без фильтров и с ценой $0.

Исправление: фильтровать тиры с `modules.length === 0` из `uniqueAlternatives` до рендера.

## Критерии приёмки

- [ ] Тир с пустым `modules` не появляется в «Other Options»
- [ ] Если после фильтрации альтернатив не остаётся — блок «Other Options» не рендерится (поведение уже обеспечено существующим `uniqueAlternatives.length > 0`)
- [ ] Primary результат не затронут — если движок выбрал primary тир с нулевыми модулями, поведение остаётся прежним (это отдельный edge-case)
- [ ] Дедупликация по `moduleKey` применяется после фильтрации пустых тиров

## Ограничения

- Не трогать primary-ветку рендера
- Не менять логику движка — фильтрация только на уровне UI

## Связи

- Зависит от: REQ-12 (дедупликация альтернатив), REQ-13 (удаление шага budget)
