# REQ-12: Hide Duplicate Alternatives in Other Options

**Статус:** Approved  
**Приоритет:** P1  
**Компонент:** configurator  

## Описание

В блоке «Other Options» (`ResultPanel`) показываются все три тира. Если жадный алгоритм с разными бюджетами выдаёт одинаковый набор модулей (например, medium и high оба выбирают RO + Sediment за $173), пользователь видит дублирующиеся карточки — это вводит в заблуждение.

Альтернативный тир скрывается, если его набор модулей совпадает с набором модулей **primary** тира (сравнение без учёта порядка).

## Критерии приёмки

- [ ] AltCard не отображается, если `new Set(alt.modules)` совпадает с `new Set(primary.modules)` (одни и те же ID, любой порядок)
- [ ] Если все альтернативы идентичны primary — блок «Other Options» скрывается целиком
- [ ] Сравнение ведётся относительно **текущего активного** primary (при переключении тира пересчитывается)
- [ ] Тир с другим набором модулей (даже одним отличием) показывается как обычно

## Ограничения

- Изменения только в `ResultPanel.tsx` — логика сравнения не выносится в движок
- Не скрывать тир из-за разной стоимости или разного покрытия — только набор модулей
