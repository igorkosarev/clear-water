# REQ-16: Кликабельные бейджи загрязнителей на карточках Systems

**Статус:** Approved  
**Приоритет:** P2  
**Компонент:** ui  

## Описание

На карточках страницы `/systems` нужно показать бейджи загрязнителей, которые система убирает (`targetContaminants` из `system-templates.json`). При клике на бейдж открывается модальное окно с деталями загрязнителя — то же, что используется в конфигураторе (REQ-10).

## Реализация

Компонент `RemainingBadges` заточен под "оставшиеся" загрязнители (label "Still in water" и т.д.). Для Systems нужен аналог с нейтральным label "Removes" / "Что убирает".

Вариант: создать общий `ContaminantBadges` компонент с пропом `label` и переиспользовать `ContaminantDetailModal`. Либо принять проп `labelKey` в `RemainingBadges`.

Выбор реализации — на усмотрение при имплементации. Дублировать логику не нужно.

## Где отображать

- На карточках страницы списка `/systems` (index.tsx) — под описанием и бюджет-бейджем
- На странице детали `/systems/:id` — не требуется (уже есть SimulationCanvas)

## Данные

`targetContaminants: string[]` в `system-templates.json` — это `ContaminantId[]`. Бейджи берут цвет, иконку и имя из `contaminants.json` по id.

## Критерии приёмки

- [ ] На каждой карточке Systems отображаются цветные бейджи загрязнителей из `targetContaminants`
- [ ] Клик по бейджу открывает `ContaminantDetailModal` с полной информацией
- [ ] Стиль бейджей идентичен конфигуратору (цвет из `contaminant.color`, иконка + название)
- [ ] Логика `ContaminantDetailModal` не дублируется — переиспользуется существующий компонент
- [ ] Label над бейджами отличается от "Still in water" (например, "Removes" / "Что убирает")

## Ограничения

- Не трогать `RemainingBadges` так, чтобы сломать конфигуратор
- Не менять структуру данных
