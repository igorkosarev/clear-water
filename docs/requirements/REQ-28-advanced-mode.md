# REQ-28: Advanced Mode — режим детального ввода для опытных пользователей

**Статус:** Done  
**Приоритет:** P2  
**Компонент:** configurator | ui | i18n  

## Описание

Существующий wizard оптимизирован под простого пользователя: симптомы, источник, бюджет. Опытный пользователь (после анализа воды, с лаборатории) хочет указать конкретные загрязнители напрямую — без прохождения шага симптомов.

**Simple Mode** (по умолчанию): текущий flow без изменений.  
**Advanced Mode** (опциональный): альтернативный flow для прямого ввода загрязнителей.

**Advanced Mode flow:**

1. Источник воды (общий с Simple Mode)
2. Назначение воды (общий с Simple Mode)
3. Статус тестирования (общий, REQ-26)
4. **Прямой выбор загрязнителей** с указанием статуса (`confirmed` / `suspected`) — вместо симптомов
5. Бюджет (общий)
6. Масштаб установки (REQ-30)

**Переключение режимов:**

- Кнопка/ссылка «Advanced Mode» на шаге симптомов или на первом экране wizard.
- При переключении в Advanced Mode шаг симптомов пропускается.
- При переключении обратно в Simple Mode — очистить прямо выбранные загрязнители.

## Что нужно сделать

1. Добавить `mode: 'simple' | 'advanced'` в состояние wizard (не в `WaterInput`).
2. В Advanced Mode заменить шаг «симптомы» на шаг «выбор загрязнителей + статус».
3. Шаг Advanced contaminants: список всех загрязнителей из БД, checkbox + radio `confirmed/suspected`.
4. Кнопка переключения режима видна на шаге симптомов / начальном экране.
5. `WizardInput` (состояние формы) расширяется: `contaminantEntries: ContaminantEntry[]` (используется в Advanced Mode вместо symptom inference).
6. Добавить i18n-ключи: `wizard.advancedMode`, `wizard.simpleMode`, `wizard.contaminantStatus.confirmed`, `wizard.contaminantStatus.suspected`.

## UI-решения

- Advanced Mode помечен визуально (badge «Advanced» или другой индикатор).
- В списке загрязнителей группировка по категории: Biological / Chemical / Physical / Radiological.
- `confirmed` отображается зелёной меткой, `suspected` — жёлтой.
- Поиск/фильтр по названию загрязнителя в списке.

## Критерии приёмки

- [ ] Simple Mode работает идентично текущей реализации
- [ ] Переключатель режима виден в wizard
- [ ] В Advanced Mode шаг симптомов заменён шагом выбора загрязнителей
- [ ] Каждый загрязнитель в Advanced Mode может быть помечен `confirmed` или `suspected`
- [ ] `ContaminantEntry[]` корректно передаётся в `runSimulation`
- [ ] Группировка загрязнителей по категории в Advanced Step
- [ ] Переключение Simple ↔ Advanced сбрасывает соответствующие данные

## Ограничения

- Simple Mode не меняется — без регрессий
- Advanced Mode не требует прохождения шага симптомов
- Режим хранится только в состоянии wizard, не в `WaterInput`
- Не добавлять новые contaminant IDs — только из существующей БД
