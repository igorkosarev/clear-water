# REQ-17: Редизайн блока «5 шагов» на intro-экране конфигуратора

**Статус:** Draft  
**Приоритет:** P2  
**Компонент:** configurator | ui  

## Описание

Блок «5 шагов» на intro-экране конфигуратора (`/configurator`, фаза `intro`) сейчас выглядит как строка иконок с шевронами между ними. Нужно переделать в полноценные карточки с акцентными номерами и hover-анимацией.

## Дизайн

**Карточка:**
- Акцентный badge с номером шага (цветной, согласно `step.color`) — отдельный элемент в верхней части карточки, не watermark
- Иконка шага
- Название (`configurator.progress.${key}`)
- Краткое описание (`configurator.progressHint.${key}`)
- Тёмный фон с тонкой границей (`border-slate-800`)

**Hover-эффект (framer-motion `whileHover`):**
- **Lift** — `translateY(-6px)` + усиленная тень
- **Glow** — `box-shadow` в цвете иконки шага (`step.color` с opacity ~0.25–0.35)

**Расположение:**
- Mobile: 1 колонка (карточки во всю ширину, стекированы вертикально)
- Desktop (sm+): 5 карточек в ряд равной ширины

**Соединители между шагами** (ChevronRight) — убрать, карточки говорят сами за себя.

## Данные

Массив `STEPS` в `Configurator.tsx` уже содержит всё нужное:
```ts
const STEPS = [
  { Icon: Waves,       color: '#38bdf8', key: 'source'     },
  { Icon: ShieldAlert, color: '#f43f5e', key: 'problems'   },
  { Icon: Utensils,    color: '#a78bfa', key: 'use'        },
  { Icon: Gauge,       color: '#fb923c', key: 'pressure'   },
  { Icon: DollarSign,  color: '#10b981', key: 'preference' },
]
```

i18n-ключи `configurator.progress.*` и `configurator.progressHint.*` уже существуют.

## Критерии приёмки

- [ ] Блок «5 шагов» отображается как 5 карточек (не строка с шевронами)
- [ ] Каждая карточка имеет цветной badge с номером шага
- [ ] Hover: карточка поднимается на ~6px и получает цветной glow в цвете шага
- [ ] На мобайле карточки стекированы в 1 колонку
- [ ] На sm+ — 5 карточек в одну строку
- [ ] Анимация через framer-motion (не CSS-only)
- [ ] i18n-ключи те же, новых не добавлять

## Ограничения

- Чисто клиентский компонент, никакого бэкенда
- Не менять `STEPS` массив и существующие i18n-ключи
- Не трогать логику переходов между фазами (intro → wizard)
