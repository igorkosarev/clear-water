# Requirement: FilterUnit + SimulationCanvas — визуализация фильтрационной системы

## What

Построить компоненты `FilterUnit`, `FilterChain` и `SimulationCanvas` — визуальное представление цепочки фильтров с анимацией потока воды и исчезновением частиц-загрязнителей по мере прохождения через фильтры. Разместить на странице `/configurator` как демо с захардкоженной системой.

## Why

Это визуальное ядро продукта: пользователь должен видеть не абстрактный список фильтров, а реальную систему — как вода проходит через каждый модуль и становится чище. Образовательный эффект достигается именно через анимированную симуляцию.

## Layout и визуальный дизайн

### Общая компоновка (top-to-bottom)

```
┌─────────────────────────────────────┐
│  [Inlet] Вода на входе              │  ← цветные частицы-загрязнители
│         ║ (вертикальная труба)      │
│  ┌──────╨────────────────────────┐  │
│  │  ≡≡  Sediment 5μm  ≡≡≡≡≡≡≡≡  │  │  ← горизонтальный корпус фильтра
│  └──────╥────────────────────────┘  │
│         ║  (particles disappear)    │
│  ┌──────╨────────────────────────┐  │
│  │  ≡≡  Activated Carbon  ≡≡≡≡≡  │  │
│  └──────╥────────────────────────┘  │
│         ║                           │
│  [Outlet] Вода на выходе           │  ← только неудалённые частицы
└─────────────────────────────────────┘
```

### FilterUnit — горизонтальный корпус фильтра
- Форма: вытянутый прямоугольник с сильно скруглёнными торцами (`border-radius` ~50% по высоте) — имитация горизонтально лежащего цилиндрического корпуса
- Левый торец — чуть темнее (крышка/head), правый — чуть светлее (дно)
- Цвет зависит от `FilterType` (см. таблицу ниже)
- Внутри: иконка типа фильтра (Lucide) слева + название по центру + текстура штриховки (opacity-низкая)
- Ширина: фиксированная (~ `w-72` / `w-80`), высота: `h-16 md:h-20`
- Вход/выход: маленькие «патрубки» сверху и снизу корпуса (или в центре — на усмотрение разработчика)

### Трубка между фильтрами
- Вертикальный сегмент `~h-12` с анимацией потока воды
- Визуально: тёмный полупрозрачный «рукав», по которому движутся цветные частицы-загрязнители
- Ширина трубки `~w-3` (тонкая)

### Анимация частиц-загрязнителей
- Каждый загрязнитель = цветная круглая частица (цвет из `contaminants.json`)
- Частицы рождаются в inlet и плывут вниз по трубке (бесконечный loop, как на Hero)
- При входе в фильтр, который их убирает (`module.removes.includes(contaminant.id)`): частица анимирует fade-out + scale(0) внутри корпуса фильтра
- После фильтра в трубке остаются только неудалённые частицы
- Частицы расположены с горизонтальным оффсетом (не все по центру) для реалистичности

## Acceptance criteria

- [ ] `FilterTypes.ts` экспортирует конфиг `FILTER_TYPE_CONFIG: Record<FilterType, { color: string, label: string, icon: LucideIcon }>`
- [ ] `FilterUnit` принимает пропсы `type: FilterType`, `inputContaminants: ContaminantId[]`, `outputContaminants: ContaminantId[]`, `animated?: boolean`, рендерит горизонтальный корпус с цветом и иконкой типа
- [ ] `FilterChain` принимает `modules: Module[]` и `inputContaminants: ContaminantId[]`, вычисляет `outputContaminants` для каждого этапа (последовательно фильтрует), рендерит вертикальную цепочку FilterUnit + труб
- [ ] `SimulationCanvas` рендерит inlet → FilterChain → outlet с анимированными частицами
- [ ] Частицы, убираемые конкретным фильтром, исчезают (fade-out + scale-down) при прохождении через него
- [ ] В outlet видны только загрязнители, не убранные ни одним фильтром из цепочки
- [ ] Страница `/configurator` показывает `SimulationCanvas` с захардкоженной системой `full_ro_system` (данные из `system-templates.json` + `modules.json` + `contaminants.json`)
- [ ] Захардкоженный ввод загрязнителей — все 8 из `contaminants.json`
- [ ] Компонент адаптивен: на мобильном всё помещается по ширине экрана (`max-w-xs sm:max-w-sm md:max-w-md mx-auto`)
- [ ] Анимация через Framer Motion, частицы deterministic (no `Math.random()`)
- [ ] Нет импортов React в `src/engine/` — движок остаётся чистым
- [ ] `npm run build` без ошибок TypeScript

## Open questions

- **Патрубки FilterUnit**: сверху/снизу корпуса (вертикальный flow) или сбоку с изгибом трубки? Рекомендация — сверху/снизу, это проще и нагляднее.
- **Количество частиц на загрязнитель**: 2–4 частицы на каждый тип, с разными offsets и задержками — чтобы поток выглядел живым, не маршем.
- **Скорость анимации**: достаточно медленная чтобы пользователь успевал читать (`duration ~2–3s` на один сегмент трубки).
